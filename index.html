HTML

A cheatsheet for HTML that I've learned so far. Structure, semantics, elements and more.

  • Every hHtml document should begin with a document type declaration <!DOCTYPE html>
  • Comments should be written as <--!I'm a comment-->and will be hidden from browser display
  • Html code should be written with spacing indentation to nested elements to provide better readability
  • Html stands for Hyper Text Mark Up Language
  • Attributes are content added to the opening tag of an element. They are made up of 2 parts, the name and value
  • To link to a relative path use ./ eg: "./localfile.html"
  • To link to an absolute path, use the full URL "https://www.file.jpg"
  • Make sure to use Semantic Html to make your code accesible, easy to read and for SEO

CSS

A cheatsheet for Css that I've learned so far. Specifity, styles, tags, class and ID.

  • Css means Cascading Style Sheets
  • It is possible to write Css code directly into Html using inline <style tag>
  • You must link the Css file in the <head> of the Html document by using <link href="styles.css" type="text/css" rel="stylesheet">
  • A Css declartion consists of a property (E.g. Size, color) and a value (E.g. 18px, red)
  • There are different ways to choose selectors: Tag; E.g p, h1, ul...Selects the corresponding element. Class name; E.g. .title ...Selects elements with the attribute class="title". ID; E.g. #school ...Selects the element with the attribute id="school"
  • Specificity is the order by which the browser decides which Css styles will be displayed. ID's are the most specific, then Class, then Tags
  • It is possible to require an Html element to have 2 or more Css selectors at the same time, E.g. h1 .special
  • Css supports selecting nested elements, E.g. .mainlist li
  • It's possible to add Css styles to multiple selectors at once, E.g. h1, .menu, p
  • !important can be applied to specific attributes to override any style, no matter how specific
